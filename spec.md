# GitHub Life Manager (GLM) 仕様書 (spec.md)

## 1. 概要
`GLM (GitHub Life Manager)` は、GitHub の Issue を TUI (Terminal User Interface) で効率的に管理するための Rust 製ツールです。
`ratatui` を使用した直感的な操作感と、詳細な編集が必要な場合に外部エディタ ($EDITOR) を起動するハイブリッドな設計を特徴としています。

## 2. 技術スタック
- **言語**: Rust (Edition 2021)
- **TUI フレームワーク**: `ratatui` + `crossterm`
- **GitHub API クライアント**: `octocrab` (認証は `gh auth token` を利用)
- **非同期ランタイム**: `tokio`
- **シリアライズ**: `serde`
- **エラーハンドリング**: `anyhow`
- **一時ファイル操作**: `tempfile`

## 3. システムアーキテクチャ
プロジェクトは以下のモジュールで構成されています。

- **`main.rs`**: エントリポイント。ターミナルの初期化、メインイベントループ、キー入力のグローバルな振り分けを担当。
- **`app.rs`**: アプリケーションの状態管理 (`App` 構造体)。画面遷移、リストの選択状態、フォーム入力状態などを保持。
- **`gh.rs`**: GitHub API との通信ロジックをカプセル化。トークン取得、リポジトリ/Issue のフェッチ、Issue 作成/更新を担当。
- **`ui.rs`**: `ratatui` を使用した描画ロジック。現在の画面状態に応じたレイアウト構築を担当。
- **`models.rs`**: アプリケーション固有のデータモデル (`Repository` 等) を定義。

## 4. 機能仕様

### 4.1 認証と初期化
- 起動時に `gh auth token` を実行して GitHub トークンを自動取得します。
- 初期画面では、自分にアサインされたオープンな Issue を全リポジトリから検索して表示します。

### 4.2 リポジトリ選択
- `r` キーで、アクセス可能なすべてのリポジトリ（Public/Private問わず）の一覧を表示します。
- リポジトリを選択すると、そのリポジトリに紐づく Issue 一覧に切り替わります。

### 4.3 Issue の閲覧と操作
- **一覧表示**: 左パネルに Issue タイトルを表示。`j`/`k` または矢印キーで上下移動。
- **詳細表示**: 右パネルに選択中の Issue の番号、タイトル、本文（Markdown）を表示。
- **新規作成**: `n` キーで作成フォームを開きます。タイトル入力後、本文は内蔵エディタまたは `Ctrl+E` で外部エディタを起動して編集可能です。
- **編集**: `e` キーで現在の Issue 本文を外部エディタで開き、保存して終了すると GitHub 上の Issue が更新されます。

### 4.4 外部エディタ ($EDITOR) 連携
1. 一時ファイル (`.md`) を作成し、現在の内容を書き出す。
2. 環境変数 `$EDITOR`（未設定時は `vi`）をサブプロセスとして起動。
3. エディタ終了後、ファイルを読み取り API 経由で GitHub へ反映。
4. ターミナルの Raw mode を復帰させ、TUI を再開。

## 5. UI レイアウト
画面は以下の 3 パネル構成を基本とします。

- **Sidebar (左上)**: カテゴリ表示（My Issues / Inbox 等）。
- **List (左下)**: Issue タイトルの一覧。
- **Main (右側)**: Issue の詳細内容。
- **Status (最下部)**: 操作ヘルプとコンテキスト情報。

## 6. キーバインド
- `q`: 終了
- `j` / `↓`: 下に移動
- `k` / `↑`: 上に移動
- `r`: リポジトリ選択画面へ
- `n`: 新規 Issue 作成 (Issue リスト画面のみ)
- `e`: 外部エディタで編集 (Issue リスト画面のみ)
- `Enter`: 決定 / 選択 / フォーム送信
- `Esc`: 戻る / キャンセル
- `Tab`: フォーム内のフィールド切り替え
- `Ctrl + E`: フォーム入力中に外部エディタを起動

## 7. 今後の展望
- **フィルタリング機能**: ラベルやマイルストーンによる Issue の絞り込み。
- **同期マネージャー**: 定期的なバックグラウンド同期。
- **ポモドーロ連携**: ステータスラインへのタイマー表示。
- **ローカルキャッシュ**: SQLite 等を用いたオフライン閲覧対応。
